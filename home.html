  • ⯈ Mysterious Organism

    •  The context of this project is that a fictional new organism has been found at the bottom of the ocean near hydrothermal vents and was named Pila aequor. In the code below, the returnRandBase() function returns a random DNA base of Adenine, Thymine, Cytosine, and Guanine. Since the organism is comprised of only 15 DNA bases, the function mockUpStrand() generates an array containing 15 bases. The factory function pAequorFactory() takes in two inputs, a number that must be unique to each organism and an array generated by mockUpStrand(). The .mutate() method changes a DNA base in the array as a way to simulate the Pila aequor's high rate of mutation. The .compareDNA() method takes one pAequorObject as an input and calculates the percentage of .dna that it shares in common with the current pAequor. These organisms are more likely to survive if their DNA is made up of at least 60% 'C' or 'G' bases, so the .willLikelySurvive() method returns a boolean based on whether or not this is true. The .complementStrand() method simply finds the complement of the P. Aequor's DNA strand by matching 'A's with 'T's and 'C's with 'G's. Finally, the arrayOfSurvivors() function continues to generate new DNA strands until 30 likely survivors are added to a new array that is returned.

  • ⯈ Django Delights

    •  This project is a website built using Django, a Python framework. While the design maintains simplicity, the primary focus was on functionality. The website serves as an inventory management system specifically designed for restaurant owners. It enables users to track inventory stock, expenses, revenue, and profit as purchases are made.Upon visiting the homepage, you will find limited content until you register and log in. Once authenticated, you gain access to the various models. Each page allows you to add, update, or delete items as necessary. The inventory page facilitates the addition of items, their unit prices, and current stock levels. Similarly, the menu page allows the addition of menu items along with their respective prices. Furthermore, you can conveniently update the recipes for each menu item. When a customer makes a purchase, the purchase page enables you to record the transaction. In doing so, the inventory items used in the recipes are automatically deducted from the stock on the inventory page. As an overview, the report page tracks revenue from the purchases model and expenses from the inventory model. By subtracting the two, it presents the total profit generated.
