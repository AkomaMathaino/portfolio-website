  • ⯈ Mysterious Organism

    •  The context of this project is that a fictional new organism has been found at the bottom of the ocean near hydrothermal vents and was named Pila aequor. In the code below, the returnRandBase() function returns a random DNA base of Adenine, Thymine, Cytosine, and Guanine. Since the organism is comprised of only 15 DNA bases, the function mockUpStrand() generates an array containing 15 bases. The factory function pAequorFactory() takes in two inputs, a number that must be unique to each organism and an array genarated by mockUpStrand(). The .mutate() method changes a DNA base in the array as a way to simulate the Pila aequor's high rate of mutation. The .compareDNA() method takes one pAequorObject as an input and calculates the percentage of .dna that it shares in common with the current pAequor. These organisms are more likely to survive if their DNA is made up of at least 60% 'C' or 'G' bases, so the .willLikelySurvive() method returns a boolean based on whether or not this is true. The .complementStrand() method simply finds the complement of the P. Aequor's DNA strand by matching 'A's with 'T's and 'C's with 'G's. Finally, the arrayOfSurvivors() function continues to generate new DNA strands until 30 likely survivors are added to a new array that is returned.

  • ⯈ Django Delights

    •  This website is my largest project which was created using Python's django framework. The design is fairly simple because my focus was primarily on the functionality, which is to allow a restaurant owner to keep track of inventory stock, as well as expenses, revenue, and profit as purchases are being made. Until you register for an account, there isn't much on the homepage. Once you register and are logged in, you will have access to the models. As each page allows, you can then add, update, or delete items. On the inventory page you can add items, unit price, and stock. You can add menu items and their prices on the menu page as well as update the recipes for each item. When a customer makes a purchase, it can be recorded on the purchase page. The inventory items used in the recipe will then be subtracted from their stock on the inventory page. Finally, the report page tracks the revenue from the purchases model, the expenses from the inventory model, and subtracts the two to show your total profit.
